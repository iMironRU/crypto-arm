# crypto-arm

**Встроенная подпись PDF по ГОСТ (PAdES) прямо в браузере.**
Без сервера, без backend, без загрузки файлов куда-либо — один HTML-файл и КриптоПро ЭЦП Browser plug-in.

**[→ Открыть инструмент](https://imironru.github.io/crypto-arm/)**

---

## Что это

`index.html` — одностраничный инструмент, который встраивает электронную подпись по ГОСТ Р 34.10-2012 прямо в структуру PDF (PAdES / CAdES-BES detached), полностью на стороне клиента. Документ подписывается и собирается заново в браузере, серверная часть не участвует вообще.

На странице виден и юридический смысл подписи: на PDF рисуется читаемый оттиск (ГОСТ Р 7.0.97) с номером сертификата, владельцем, сроком действия и датой подписания — а не только невидимая криптография внутри файла.

## Возможности

- **Видимый оттиск подписи** — заголовок, сертификат, владелец, срок действия, дата подписания
- **Несколько подписей на одном документе** — можно подписать один файл сразу несколькими сертификатами, штампы не перекрывают друг друга
- **Пакетная подпись** — до 5 PDF за один проход; на выходе один PDF или `signed.zip`, если файлов больше одного
- **Инкрементальные обновления PDF** — повторная подпись дописывается новой ревизией и не портит уже стоящие подписи (как это делают Adobe/PDFBox)
- **Диагностика плагина** — если КриптоПро не установлен или не откликается, показывает точную причину и ссылки на установку CSP / плагина / расширения браузера
- **Автообновление кеша** — страница сама подсказывает, когда вышла новая версия, без Ctrl+Shift+R

## Как пользоваться

1. Открой [страницу инструмента](https://imironru.github.io/crypto-arm/)
2. Загрузи один или несколько PDF (drag-and-drop или выбором файлов)
3. Отметь один или несколько сертификатов из локального хранилища КриптоПро
4. Нажми «Подписать» — плагин запросит ПИН, документ(ы) скачаются подписанными

## Проверка результата

- [Сервис проверки подписей КриптоПро](https://saas.cryptopro.ru/verifycpca/)
- КриптоАРМ
- Adobe Acrobat (открывает PAdES «из коробки»)
- [sign.kloud.one](https://sign.kloud.one)

## Как это устроено технически

- Подпись формата **CAdES-BES detached**, встроена по правилам **PAdES** (`SubFilter /ETSI.CAdES.detached`, `/ByteRange`, `/Contents`)
- Хэш ГОСТ Р 34.11-2012 считает сам плагин КриптоПро (`CAdESCOM.HashedData` → `CAdESCOM.CadesSignedData.SignHash`), браузер только готовит байты и встраивает готовый CMS-контейнер
- PDF собирается через [pdf-lib](https://pdf-lib.js.org/) + [fontkit](https://github.com/Pdf-lib/fontkit) (кириллический шрифт для оттиска встраивается как subset)
- Вместо полного пересохранения документа используется **инкрементальное обновление**: в файл дописываются только новые/изменённые объекты (штамп, шрифт, словарь подписи, AcroForm) и новый `xref` со `/Prev` на предыдущую ревизию — существующие байты, включая уже стоящие подписи, не трогаются
- Несколько подписанных файлов пакуются в архив через [JSZip](https://stuk.github.io/jszip/)

## Требования

- Установлен **CryptoPro CSP**, **КриптоПро ЭЦП Browser plug-in** и расширение-мост для браузера
- Сертификат ГОСТ в личном хранилище пользователя (`CurrentUser \ My`)
- Страница подключает `cadesplugin_api.js` с `cryptopro.ru` — нужен доступ к этому домену

## Запуск локально

Сборка не требуется — это один статический HTML-файл:

```bash
python3 -m http.server 8080
# открыть http://localhost:8080/
```

Для закрытого контура без доступа в интернет можно скачать `cadesplugin_api.js` с официального сайта КриптоПро и подключить локально — см. комментарий в начале файла.

## Ограничения

- Зашифрованные / защищённые паролем PDF не поддерживаются
- Алгоритмы хеширования — только ГОСТ Р 34.11-2012 (256/512) и ГОСТ Р 34.11-94, по OID сертификата
- Работает только в браузере с установленным и разрешённым расширением КриптоПро (см. баннер диагностики на странице)
